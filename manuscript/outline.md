# 상세 목차

## 1. 왜 어려운가
- Tistory Open API 종료(2024-02)와 그 여파
- "API 발행" vs "브라우저 발행"의 근본적 차이
- 자동 블로그가 멈추는 6가지 지점(세션·봇탐지·프로필락·멈춤·중복·고갈)

## 2. 콘텐츠 엔진
- Gemini로 검색되는 글 생성: 프롬프트 설계
- 검색어 타겟팅(독자가 실제 검색하는 쿼리를 본문에 반영)
- SEO 제목·메타 스니펫(검색결과 첫 문단) 최적화

## 3. 발행 엔진
- undetected-chromedriver로 봇탐지 우회의 기본
- Tistory 에디터(TinyMCE) 자동 조작: 본문·이미지·태그
- 로그인 3단계(프로필 쿠키 → 세션 주입 → 카카오 로그인)

## 4. 안정화 (핵심)
- anti-bot 대응: DKAPTCHA 등 도전과제 처리 원리
- 세션/2FA: 재로그인 없이 세션 유지하는 법
- 1회용 임시 프로필: 공유 프로필 버그 클래스를 구조적으로 제거
- 멈춤 자가복구: 하드 타임아웃·워치독·락 잔여물 정리

## 5. 무인 운영
- 스케줄링: 수집/발행 잡 분리
- 내구성 발행 큐: enqueue/worker 분리, 백오프 재시도, dead-letter
- 중복 발행 차단: 멱등 가드

## 6. 색인·수익화 현실
- 구글 색인(Indexing API)·네이버 서치어드바이저 등록의 실제
- 광고(AdSense/AdFit)·CPA 배선 구조
- **솔직한 숫자 공개**: 트래픽·실수익(현재 ₩0)과 그 원인 분석 — 무엇이 병목인가
